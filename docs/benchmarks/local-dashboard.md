# Local Benchmark Dashboard

Benchmarks run locally on a trusted Apple Silicon Mac so CoreML and MLX measurements reflect the actual hardware without registering a public CI runner.

## Running

Use `scripts/run_benchmarks.sh` from the repository root. It builds `asr-bench`, builds the MLX metallib, downloads LibriSpeech `test-clean` when needed, runs the benchmark, and records a timestamped result directory.

Default ASR engines:

- `qwen3-mlx-0.6b-8bit`
- `qwen3-mlx-0.6b-4bit`
- `parakeet`
- `omnilingual-mlx-300m-4bit`

Default run:

```bash
scripts/run_benchmarks.sh
```

Short smoke run:

```bash
scripts/run_benchmarks.sh \
  --limit 10 \
  --engines "qwen3-mlx-0.6b-4bit parakeet"
```

Full LibriSpeech run for published-table comparisons:

```bash
scripts/run_benchmarks.sh --full
```

Run VAD or diarization benchmarks from local manifests:

```bash
scripts/run_benchmarks.sh \
  --skip-asr \
  --vad-manifest .benchmark-datasets/vad/manifest.tsv \
  --vad-engines "silero-coreml silero-mlx firered pyannote"

scripts/run_benchmarks.sh \
  --skip-asr \
  --diarization-manifest .benchmark-datasets/diarization/manifest.tsv \
  --diarization-engines "sortformer-default pyannote-mlx"
```

Useful options:

- `--dataset <path>` uses an existing LibriSpeech-style directory or TSV manifest.
- `--dataset-root <path>` changes where LibriSpeech is downloaded.
- `--results-root <path>` changes where result JSON and the dashboard are written.
- `--full` runs the entire dataset instead of the default 200-utterance local run.
- `--skip-asr` skips ASR when running only VAD or diarization.
- `--vad-manifest <path>` runs VAD metrics from a local manifest.
- `--vad-engines "<ids>"` sets VAD engines: `silero-coreml`, `silero-mlx`, `firered`, `pyannote`.
- `--vad-silero-onset <float>` and `--vad-silero-offset <float>` tune Silero VAD thresholds. The local runner defaults to `0.25`/`0.20`, which improved span F1 on the mini50 comparison set while preserving file-level classification.
- `--vad-silero-min-speech <seconds>` and `--vad-silero-min-silence <seconds>` tune Silero post-processing durations.
- `--diarization-manifest <path>` runs diarization metrics from a local manifest.
- `--diarization-engines "<ids>"` sets diarization engines: `sortformer-default`, `sortformer-balanced`, `sortformer-streaming`, `pyannote-mlx`, `pyannote-coreml`.
- `--force-download` deletes and re-downloads LibriSpeech `test-clean`.
- `--prepare-only` builds tools and prepares the dataset without running inference.

VAD and diarization manifests use tab-separated paths:

```text
audio/path.wav	reference/path.rttm	optional-id
```

Reference files can be RTTM. VAD also accepts plain text files with `start end`
per line. Diarization also accepts plain text files with `start end speaker`
per line.

## Outputs

Each run writes:

- `asr.json` from `asr-bench`, containing aggregate WER/CER, average and median per-file WER/CER, substitutions, insertions, deletions, mean/median/overall xRT, cold-load time, and peak RSS per engine.
- `vad.json` from `vad-bench`, containing frame/span F1, file-level F1, precision, recall, false alarm rate, miss rate, mean/median/overall xRT, load time, and peak RSS per engine.
- `diarization.json` from `diarization-bench`, containing DER, JER-style speaker overlap error, miss, false alarm, speaker error, speaker-count accuracy, mean/median/overall xRT, load time, and peak RSS per engine.
- `asr.log`, containing the benchmark console output.
- `metadata.json`, containing commit SHA, branch, host, macOS version, CPU, dataset, and benchmark configuration.
- `index.html`, a static dashboard generated by `scripts/render_benchmark_dashboard.py`.

Results are stored under:

```text
benchmark-results/
  index.html
  runs/
    <timestamp>-<sha>/
      asr.json
      vad.json
      diarization.json
      asr.log
      vad.log
      diarization.log
      metadata.json
```

The dashboard renderer keeps the newest 90 runs by default.

## Extending

Add new benchmark families only after the command can emit deterministic JSON with enough metadata to compare runs. The expected shape is:

- one directory per local run under `benchmark-results/runs/<timestamp>-<sha>/`
- one `metadata.json` sidecar
- one result JSON per benchmark family, for example `vad.json`, `diarization.json`, `tts.json`, or `align.json`

Useful benchmark families to add next:

- VAD: precision, recall, F1, false alarm rate, miss rate, RTF, load time, peak RSS.
- Diarization: DER, JER, miss, false alarm, speaker error, speaker-count accuracy, RTF, peak RSS.
- TTS: output duration, wall time, RTF/xRT, time to first audio, peak RSS, and round-trip WER with a fixed ASR engine.
- Forced alignment: timestamp error on a fixed fixture, span validity, wall time, RTF, peak RSS.
