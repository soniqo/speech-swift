# Daily Benchmark Dashboard

The daily benchmark workflow runs on a real macOS ARM64 runner so CoreML and MLX measurements reflect Apple Silicon hardware rather than a virtualized GitHub-hosted runner.

## Workflow

`.github/workflows/daily-benchmarks.yml` runs every day at 03:30 UTC and can also be launched manually. The first lane benchmarks ASR with the existing `asr-bench` executable against LibriSpeech `test-clean`.

Default ASR engines:

- `qwen3-mlx-0.6b-8bit`
- `qwen3-mlx-0.6b-4bit`
- `parakeet`
- `omnilingual-mlx-300m-4bit`

Manual dispatch inputs let maintainers change the runner labels, ASR utterance limit, engine list, and whether to publish the dashboard.

## Outputs

Each run writes:

- `asr.json` from `asr-bench`, containing WER, substitutions, insertions, deletions, RTF, xRT, cold-load time, and peak RSS per engine.
- `metadata.json`, containing commit SHA, branch, runner, macOS version, CPU, dataset, and benchmark configuration.
- `index.html`, a static dashboard generated by `scripts/render_benchmark_dashboard.py`.

The workflow uploads the run directory as an Actions artifact and, by default, copies the newest run into `gh-pages:/benchmarks/data/runs/`. The dashboard renderer keeps the newest 90 runs and updates `gh-pages:/benchmarks/index.html`.

## Extending

Add new daily benchmark lanes only after the command can emit deterministic JSON with enough metadata to compare runs. The expected shape is:

- one directory per workflow run under `benchmark-results/runs/<timestamp>-<sha>/`
- one `metadata.json` sidecar
- one result JSON per benchmark family, for example `vad.json`, `diarization.json`, `tts.json`, or `align.json`

Useful benchmark families to add next:

- VAD: precision, recall, F1, false alarm rate, miss rate, RTF, load time, peak RSS.
- Diarization: DER, JER, miss, false alarm, speaker error, speaker-count accuracy, RTF, peak RSS.
- TTS: output duration, wall time, RTF/xRT, time to first audio, peak RSS, and round-trip WER with a fixed ASR engine.
- Forced alignment: timestamp error on a fixed fixture, span validity, wall time, RTF, peak RSS.
